# OneAPI_FPGA

## Project Goal
To implement a systolic array for dense matrix multiplication using Intel OneAPI, 
and analyse the effect of varying pipeline properties and matrix size on latency and fMAX using 
the OneAPI optimization report.

## What This Project Does
This project implements matrix multiplication on FPGA using a systolic array 
architecture. Pipeline performance is analysed by examining the optimization 
report generated by the Intel OneAPI compiler.

## Key Concepts

**FPGA (Field-Programmable Gate Array)**: A reconfigurable hardware chip whose 
logic circuits can be defined through code.

**Systolic Array**: A grid of processing elements (PEs) where data flows 
rhythmically between neighbours. Each PE performs a multiply-accumulate 
operation independently, enabling high parallelism.

**Pipeline**: A technique where multiple operations are overlapped in execution, 
similar to an assembly line.

**II (Initiation Interval)**: A property of a pipeline. The number of clock 
cycles before a new loop iteration can begin. II=1 is ideal.

**Latency**: The number of clock cycles required to complete a computation.

**fMAX**: The maximum clock frequency (MHz) at which the FPGA circuit can operate.

**FPGA Emulation** (`make fpga_emu`): Runs the algorithm on the CPU for 
bit-accurate verification of design correctness.

**FPGA Report** (`make report`): Generates a static analysis of how the design 
maps to FPGA, providing estimates of II, latency, and fMAX without running on 
hardware.

**FPGA Simulation** (`make fpga_sim`): Simulates the design on the actual FPGA 
chip. Requires Quartus/ModelSim licences.

**FPGA Build** (`make fpga`): Builds and deploys firmware onto a physical FPGA. 
The only mode where wall-clock time measurement is meaningful.
